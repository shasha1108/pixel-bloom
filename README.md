## рҹ“– About

**pixel-bloom** is a Claude Code skill that generates complete, standalone HTML pages вҖ” each one a living pixel-art world wrapped in luminous Frutiger Aero glass. It combines five disciplines into a single output:

| Pillar | Role |
|--------|------|
| рҹҺЁ **p5.js pixel rendering** | Sharp, grid-aligned sprites вҖ” zero anti-aliasing, pure pixel aesthetic |
| рҹӘҹ **Frutiger Aero glassmorphism** | Frosted glass shells, ambient light orbs, Ganzfeld light fields |
| рҹҢҝ **Procedural flora** | 4 algorithmic plant models that grow, sway, and breathe |
| рҹҗҹ **AI FSM creatures** | Wander / Chase / Flee state machines driven by Perlin noise |
| рҹ”Ҡ **Web Audio synthesis** | Healing bowls, ambient drones, binaural beats вҖ” code only, zero audio files |

> **The output is a single `.html` file.** No build step, no framework, no dependencies beyond p5.js CDN. Open it in any browser and it runs.

## вҡҷпёҸ How It Works

### The Generation Pipeline (6 Steps)

```
STEP 0           STEP 1вҖ“3          STEP 4            STEP 4.5       STEP 5вҖ“5.5        STEP 6
Visual           Design            Code              Polish         Quality           Deliver
Research  в”Җв”ҖвҶ’    Decisions  в”Җв”ҖвҶ’    Generation  в”Җв”ҖвҶ’   (Refine)  в”Җв”ҖвҶ’  Assurance  в”Җв”ҖвҶ’   & Verify
(new scenes)     Canvas ratio      From defensive    Delete only    validate.py       Browser test
                 Scene mode        skeleton          Micro-tune     Reader test       Touch check
                 Audio profile     Templates         Strengthen     Emotional check   Format output
                 Decompose scene   Flora models      resonance
                 Tech decisions    FSM code
```

<details>
<summary><strong>рҹ”Қ Step-by-step details</strong></summary>

| Step | What Happens |
|------|-------------|
| **STEP 0** | Visual research вҖ” for new scene types, gather reference on real-world glass, plants, creature behaviors |
| **STEP 1** | Decide canvas ratio, audio profile, and scene mode (closed/open/Ganzfeld) |
| **STEP 2** | Decompose the user's description into pixel elements + assign procedural models |
| **STEP 3** | 5 technical decisions: canvas type, motion mode, glass type, particle count, audio profile |
| **STEP 4** | Code generation from defensive skeleton + matching flora/FSM templates |
| **STEP 4.5** | Polish вҖ” *only* delete, micro-tune parameters, strengthen resonance. No new entities |
| **STEP 5** | Run `validate.py` вҖ” 8 categories, must pass all green |
| **STEP 5.5** | **Reader test** вҖ” shift perspective to a first-time user: does the first second feel right? |
| **STEP 6** | Browser verification via `/verify`, touch check, final delivery |

</details>

### Immutable Architecture: The Z-Index Sandwich

Every generated page uses this exact layer stack вҖ” never changed:

```
z=1  рҹҢҹ  Ambient Light Orbs       CSS radial-gradient, fixed, slow float
z=2  рҹӘҹ  Frosted Glass Base       ONLY layer with backdrop-filter: blur()
z=3  рҹҺЁ  Pixel Canvas             p5.js render surface, drop-shadow
z=4  вңЁ  Glass Shell              ::after Fresnel highlight, NO blur, pointer-events: none
z=5  рҹ–җпёҸ  Interaction Intercept    Native pointerdown events, no synthetic clicks
```

### Three Biomimetic Motion Laws

| Law | Principle | Implementation |
|-----|-----------|---------------|
| рҹ”„ **Life Breathing** | No object is ever perfectly still | `sin`/`cos` micro-motion on every entity |
| рҹҢҠ **Soul Wandering** | Smooth exploration, not mechanical patrol | Perlin noise (`noise()`) вҖ” never `random()` for movement |
| рҹҚ® **Delayed Gratification** | Weight and "jelly-like" feel | `lerp()` damping on all transitions |

## рҹ”¬ Quality Standards

pixel-bloom enforces a multi-layered quality gate before any output is delivered:

| Gate | Tool | Checks |
|------|------|--------|
| рҹ”§ **Structural** | `scripts/validate.py` | Meta headers, p5.js pixel rules (`pixelDensity(1)`, `noSmooth()`), z-index sandwich integrity, JS syntax (`node --check`) |
| рҹҺЁ **Aesthetic** | Manual review | Color discipline (no dark mode, no high-saturation primaries), Frutiger Aero palette adherence |
| рҹ§  **Interaction** | Manual review | Click/double-click separation (300ms debounce), mobile touch (no default scroll), drag support |
| рҹ”Ҡ **Audio** | Manual review | AudioContext resume on first interaction, volume envelopes, no audio file dependencies |
| вқӨпёҸ **Emotional** | **Reader Test** | 4 questions: first-second feeling, instinctive interaction targets, 10-second retention, "AI-made" tells |

> The **Reader Test** (STEP 5.5) is unique to pixel-bloom: the AI must shift perspective and evaluate the output as a first-time user, checking emotional resonance against the concept seed вҖ” not just technical correctness.
